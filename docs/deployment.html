A.2 Docker 化

最小 Dockerfile

FROM python:3.12-slim

WORKDIR /app
COPY mini_agent/ ./mini_agent/
COPY requirements.txt .

RUN pip install --no-cache-dir -r requirements.txt

EXPOSE 3000

ENV PYTHONUNBUFFERED=1

CMD ["python", "-m", "mini_agent"]

requirements.txt

openai>=1.0
anthropic>=0.40
croniter>=2.0

构建 & 运行

docker build -t mini-agent .

# Mock 模式
docker run -it --rm -p 3000:3000 mini-agent

# 真实 LLM
docker run -it --rm -p 3000:3000 \
  -e OPENAI_API_KEY=sk-xxx \
  mini-agent

Docker Compose（推荐）

services:
  agent:
    build: .
    ports:
      - "3000:3000"
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
    volumes:
      - ./data:/app/data
      - ./skills:/app/skills
    restart: unless-stopped

docker compose up -d
docker compose logs -f agent

A.3 VPS 部署（Ubuntu/Debian）

步骤 1：准备环境

sudo apt update && sudo apt install -y python3.12 python3.12-venv git

步骤 2：部署代码

cd /opt
git clone <your-repo> mini-agent
cd mini-agent
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

步骤 3：systemd 服务

# /etc/systemd/system/mini-agent.service
[Unit]
Description=Mini Agent - Six-Layer AI Assistant
After=network.target

[Service]
Type=simple
User=agent
WorkingDirectory=/opt/mini-agent
Environment=OPENAI_API_KEY=sk-xxx
ExecStart=/opt/mini-agent/.venv/bin/python -m mini_agent
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target

sudo systemctl daemon-reload
sudo systemctl enable --now mini-agent
sudo journalctl -u mini-agent -f

A.5 日志与监控

结构化日志

mini_agent 的 main.py 使用 Python logging 模块。生产环境建议改为 JSON 格式：

import json, logging

class JSONFormatter(logging.Formatter):
    def format(self, record):
        return json.dumps({
            "ts": self.formatTime(record),
            "level": record.levelname,
            "logger": record.name,
            "msg": record.getMessage(),
        })

handler = logging.StreamHandler()
handler.setFormatter(JSONFormatter())
logging.root.addHandler(handler)

审计日志分析

audit.jsonl 记录了每次工具调用的完整轨迹：

# 统计工具调用频次
cat audit.jsonl | python -c "
import json, sys, collections
tools = collections.Counter(
    json.loads(l)['tool'] for l in sys.stdin)
for t, c in tools.most_common():
    print(f'{t}: {c}')
"

# 查看失败的调用
cat audit.jsonl | python -c "
import json, sys
for l in sys.stdin:
    e = json.loads(l)
    if not e['success']:
        print(f\"{e['tool']}: {e['error']}\")
"

A.6 安全加固清单

部署到公网前的最低安全要求：

  • [ ] API Key 不要硬编码——使用环境变量或 secrets manager
  • [ ] HTTP 端口加认证——mini_agent 的 HTTP 渠道默认无认证，需添加 token 验证
  • [ ] 限制工具白名单——生产环境中 allowed_tools 不要用 ["*"]
  • [ ] 设置 denied_tools——至少禁止 run_command、delete_file
  • [ ] 审计日志定期归档——audit.jsonl 会持续增长
  • [ ] 设置 LLM 调用预算——通过 OpenAI 的 usage limits 防止意外费用

A.6 Security Hardening Checklist

Minimum security before exposing to the internet:

  • [ ] No hardcoded API keys — use environment variables or a secrets manager
  • [ ] Add auth to the HTTP channel — mini_agent's HTTP channel has no auth by default; add token validation
  • [ ] Restrict tool whitelist — in production, avoid allowed_tools=["*"]
  • [ ] Configure denied_tools — at minimum block run_command, delete_file
  • [ ] Rotate and archive audit logs — audit.jsonl grows indefinitely
  • [ ] Set LLM spend limits — use OpenAI usage limits to avoid billing surprises
